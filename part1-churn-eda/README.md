# Part 1 — Data Audit, EDA & Business Understanding

Capstone: D2C Customer Churn Intelligence & Retention API

---

## What's in this repo

```
├── eda_audit.ipynb          # Main analysis notebook (load → QA → EDA → hypotheses)
├── data_quality_report.md   # Structured report on all data issues found
├── business_memo.md         # Business-facing memo with findings and recommendations
├── requirements.txt         # Python dependencies
├── data/                    # Put the 7 CSV files here (see below)
└── charts/                  # Auto-generated by the notebook
```

---

## Setup

```bash
pip install -r requirements.txt
```

---

## Data

Download the dataset from the Google Drive link provided in the problem statement.  
Place all 7 CSV files in the `data/` folder:

```
data/customers.csv
data/orders.csv
data/support_tickets.csv
data/web_events_snapshot.csv
data/churn_labels.csv
data/rfm_modeling_snapshot.csv
data/intervention_history.csv
```

---

## Run the notebook

Open `eda_audit.ipynb` in VS Code or JupyterLab and run all cells top to bottom.  
Charts will be saved to the `charts/` folder automatically.

```bash
jupyter notebook eda_audit.ipynb
```

---

## Key findings (summary)

- Overall churn rate: **47%**
- Customers with last order > 90 days ago churn at **83%**
- Customers with zero web sessions in last 30 days churn at **66%**
- 57.8% of customers are not enrolled in the loyalty programme
- 12 duplicate order records (`_DUP` suffix) and 1,872 post-snapshot rows identified
- Discount-heavy buyers show elevated churn risk
